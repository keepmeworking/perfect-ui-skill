# Dashboard and Data UX

Dashboards should help users decide or act, not merely display metrics.

Prioritize the most important tasks and exceptions. Use clear page titles, time ranges, data freshness, units, comparison context and actionable states.

Tables:
- Left-align text and right-align numeric data.
- Keep sort state, selected rows and bulk actions visible.
- Use sticky headers only when they improve scanning.
- Distinguish no data, no results, loading, error and permission states.
- Preserve column priority on compact layouts.

Charts:
- Include accessible titles, summaries and data-table alternatives when needed.
- Do not rely on color alone.
- Avoid misleading axes and decorative chart junk.
- Show loading, missing and stale-data behavior.

Filters should be discoverable, removable and reflected in URL/state where appropriate. Preserve user choices and provide clear reset behavior.
