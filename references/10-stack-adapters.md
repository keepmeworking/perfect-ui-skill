# Stack Adapters

## React / Next.js
Keep server and client boundaries deliberate. Use semantic components, route-level loading/error boundaries, stable keys, optimized images and accessible headless primitives where appropriate.

## Tailwind
Map utilities to a project token theme. Avoid repeated arbitrary values. Extract reusable components by behavior, not merely class length.

## Vue / Svelte
Keep state and accessibility behavior inside reusable components. Preserve native semantics and test transitions with reduced motion.

## React Native
Use platform-safe areas, dynamic type, accessibility roles/labels, keyboard avoidance, reduced motion and minimum touch targets. Do not copy web interaction patterns blindly.

## Flutter
Centralize `ThemeData`, color schemes, typography and component themes. Respect text scaling, semantics, focus traversal and platform conventions.

## Native iOS / Android
Follow platform navigation, typography, touch, back behavior, permissions, safe areas and accessibility APIs while sharing semantic design tokens where practical.
